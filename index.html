Mist.io expands to KVM, vCloud and vSphere, adds OAuth support

There have been a host of new features added lately in Mist.io, so we thought it was time to send you an update in case you missed them.

KVM, vCloud and vSphere support

In our never ending quest to support as many workflows as possible, we've added support for KVM hypervisors. Instead of having to log in to the hypervisor to reboot a VM that has hanged, you can now do it with a few taps on your smartphone, laptop or tablet, along with the rest of your infrastructure.

Read More

Using Mist.io with your Google or Github account

We’ve also just added oAuth support in Mist.io which means you can log in using your Google or Github account. Adding oAuth support makes accessing Mist.io much easier, but there are also several security reasons behind our decision to support social authentication.

Read More

Adding your own cloud provider to Mist.io

Mist.io supports all popular cloud providers, but there are some that we didn’t get around to adding yet. Mist.io’s management platform is open source, so if your favorite provider is not supported, don’t worry because it’s fairly easy to add a new one. We’ve documented the process when we added Host Virtual, a highly flexible provider with data centers in 20 locations around the world.
